# 🔐 IAM User Lifecycle Management Lab

## Overview

This project demonstrates a simulated Identity and Access Management (IAM) User Lifecycle process aligned with banking security, audit, and compliance requirements.

The lab focuses on Joiner, Mover, and Leaver (JML) workflows using Microsoft Entra ID and RBAC-based access management principles.

The objective was to understand how organizations manage user access securely throughout the employee lifecycle while reducing insider risk, preventing privilege creep, and maintaining audit readiness.

---

## Environment

- Microsoft Entra ID (Azure AD)
- RBAC Groups
- ServiceNow (Process Simulation)
- Approval Workflows
- IAM Runbooks and SOPs

---

## IAM Processes Demonstrated

### Joiner

- User provisioning
- Group-based access assignment
- RBAC implementation
- Approval validation
- Audit verification

### Mover

- Role change management
- Access review
- Removal of outdated permissions
- Assignment of new RBAC groups
- Privilege creep prevention

### Leaver

- Account de-provisioning
- Group membership removal
- Access revocation
- Audit validation
- Insider risk reduction

---

## Security Concepts Applied

- Least Privilege
- Role-Based Access Control (RBAC)
- Segregation of Duties (SoD)
- Approval-Based Provisioning
- Access Reviews
- Audit Logging
- Governance Controls
- SLA Adherence
- Escalation Handling

---

## Key Learning

Identity security is not just about granting access.

The real challenge is ensuring access remains appropriate throughout a user's lifecycle while maintaining security, compliance, and audit requirements.
